## Troubleshooting Quick Reference

### Problem: "Line Lost" Frequently
**Solution:** Run video processor, check "Window 3: Clean Mask" panel - if black line isn't there, adjust HSV bounds in code

### Problem: Robot Oscillates
**Solution:** Reduce Kd value in robot_logic.py (was 0.001, try 0.0005)

### Problem: Robot Doesn't Turn
**Solution:** Increase Kp value in robot_logic.py (was 0.005, try 0.01)

### Problem: Can't See Pipeline Stages
**Solution:** Ensure `show_pipeline=True` in video_processor.py line 194

### Problem: Simulation Too Fast
**Solution:** Increase slow_motion_factor in simulator.py line 148 (try 3 or 4)

→ For more troubleshooting: See DEBUG_GUIDE.md

## Architecture Summary

All three modes use the **same core logic**:

```
Input (Video/Sim Camera)
    ↓
Detect black lines ← SAME ALGORITHM
    ↓
Calculate midpoint error ← SAME ALGORITHM
    ↓
Apply PD controller ← SAME ALGORITHM
    ↓
Output steering command ← SAME ALGORITHM
    ↓
Output (Motor/Sim Physics)
```

**Result:** Consistent behavior across video processor, simulator, and physical robot.
